# 一、全文大纲

------

## 摘要

- 背景：英雄联盟数据体量大、维度多、版本与赛事更新频繁
- 方法：构建一个基于本地数据的交互式可视化分析平台（VisLOL）
- 数据源：Data Dragon、Riot LoL API、Oracle’s Elixir
- 重点：以任务驱动的可视化设计支持多层级分析
- 结果：通过多个模块与图表揭示版本生态、玩家特征与职业赛事格局

------

## 第1章 引言

1. 英雄联盟生态与数据分析需求
2. 现有分析方式的局限（数据分散、复现性差、缺乏交互）
3. 本文目标：设计并实现一个面向分析任务的可视化系统
4. 主要贡献：
   - 多源数据的统一分析语境
   - 面向任务的可视化与交互设计
   - 支持版本、赛区、战队、选手等多层级探索

------

## 第2章 数据与方法

> **这一章只写“支撑可视化分析所必需的内容”**

### 2.1 数据来源与粒度

- Data Dragon：版本化静态数据（英雄 / 装备）
- Riot LoL API：玩家画像与个人对局（可选）
- Oracle’s Elixir：职业赛事逐场逐选手统计数据

### 2.2 数据预处理与一致性

- 数值字段清洗（NaN / inf 处理）
- 不同粒度行的使用策略（team 行 vs player 行）
- 缺失数据的补全与替代策略
- 统一过滤条件（years / leagues）确保分析口径一致

### 2.3 指标定义与派生

- KDA、KP、DPM、GPM
- 团队伤害占比（由选手数据聚合）
- 雷达图的 min–max 归一化方法

------

## 第3章 可视化设计与分析【核心章节】

### 3.1 可视化设计原则与分析任务

- 任务驱动设计
- 多视图协同与层级探索
- 可读性优先与聚合策略
- 可解释指标与非预测导向
- 手动更新机制与可复现性

------

### 3.2 游戏生态模块：版本、英雄与装备

#### 3.2.1 版本语境（Patch Context）

- 版本选择作为静态分析的前置条件

#### 3.2.2 英雄属性空间与成长潜力

- 英雄属性散点图
- 英雄成长 delta 对比

#### 3.2.3 装备效率与合成结构

- 装备属性效率排序
- 装备合成路径结构展示

------

### 3.3 玩家模块：个人表现与对局分析（次要模块）

#### 3.3.1 玩家画像与英雄池

- 玩家基础信息
- 英雄熟练度分布

#### 3.3.2 成就画像与近期表现

- Challenges 雷达图
- 最近对局趋势分析

------

### 3.4 电竞赛事模块：赛区、战队、选手与 Meta（重点模块）

#### 3.4.1 赛事规模与赛区风格

- 赛区比赛数量
- 比赛时长与历史趋势

#### 3.4.2 战队风格与多维画像

- 战队资源/输出风格散点
- 战队雷达图（多维能力）

#### 3.4.3 选手与位置差异分析

- 位置职责分布
- 输出与生存关系

#### 3.4.4 英雄选择与 BP Meta

- 英雄 Pick / Ban 热度
- 赛区偏好热力图
- 位置–英雄桑基图
- 英雄跨赛季趋势

------

### 3.5 小结：多层级可视化分析能力

- 从版本 → 英雄 → 装备
- 从赛区 → 战队 → 选手 → 英雄
- 可视化如何支持探索式分析

------

## 第4章 讨论与局限

- 数据层面的局限（非实时、缺字段）
- 可视化层面的局限（SVG 复杂交互能力）
- 系统扩展方向（更细粒度对局、更多数据源）

------

## 第5章 结论

- 总结 VisLOL 的设计与实现
- 强调可视化分析的价值
- 展望未来改进方向
